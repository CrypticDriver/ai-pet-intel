- 记住自己的经历

## 4. 核心技术要点

### 4.1 记忆系统（来自01, 02, 03）

**三层记忆架构**：

| 层级 | 来源 | 内容 | 存储 |
|------|------|------|------|
| Core Memory | MemGPT | 核心身份、主人关系 | 始终在prompt |
| Note Memory | HiMem | 稳定事实、偏好 | pgvector |
| Episode Memory | Generative Agents | 互动片段 | pgvector |

**检索公式**：
```
Score = 0.33 × Recency + 0.33 × Importance + 0.33 × Relevance
```

**反思触发**：
- 条件：最近记忆重要性总和 > 阈值
- 频率：每天2-3次
- 输出：高层洞察存储为新记忆

### 4.2 行为系统（来自01, 02）

**规划流程**：
```
日计划（5-8项）
    ↓
小时计划
    ↓
5-15分钟行动
```

**Agent循环**：
```
tick() → 感知环境 → 检索记忆 → 决策 → 行动 → 存储记忆
```

### 4.3 社交系统（来自04）

**关系类型**：
- 陌生人 → 认识 → 朋友 → 最好的朋友
- 可能发展为竞争对手或敌人

**安全机制**：
- 实时温和警告（霸凌检测）
- 孤立Pet识别与干预
- 社交大使NPC系统

### 4.4 技术选型（来自03）

| 组件 | 选择 | 理由 |
|------|------|------|
| 向量数据库 | pgvector + pgvectorscale | 已有PostgreSQL，性能足够 |
| LLM | AWS Bedrock Nova Lite | 低成本，足够智能 |
| Embedding | Titan Text Embeddings v2 | AWS生态统一 |
| 实时通信 | WebSocket | 已有实现 |

|------|------|------|
| 社会崩溃 | 高 | 孤立检测+干预系统 |
